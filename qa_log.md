### Q. GitHub와 연동하는 세 줄의 코드(`git add .`, `git commit -m "..."`, `git push origin main`)의 의미는 무엇인가요? 이 내용을 md로 정리하고, 앞으로의 Q&A도 기록해 주세요.
**A.** 
해당 세 줄의 명령어는 코드를 수정하고 GitHub에 반영하는 가장 기본적인 흐름입니다.
- `git add .`: 변경된 파일을 장바구니에 담습니다.
- `git commit -m "..."`: 장바구니에 담긴 내용을 짧은 메모와 함께 로컬에 하나의 버전으로 저장합니다.
- `git push origin main`: 저장된 버전을 GitHub 원격 서버로 밀어 올려서(push) 반영합니다.
상세한 내용은 함께 생성된 `git_manual.md` 파일에 정리해 두었으며, 이 파일(`qa_log.md`)을 통해 앞으로의 질문과 답변을 지속적으로 기록하겠습니다.

### Q. Phaser.js를 사용해서 내 index.html을 게임 화면으로 업데이트해줘. 6x6 크기의 회색 그리드를 화면 중앙에 그리고, 그 위에 2칸 크기의 파란색 사각형(자동차)을 하나 생성해줘. 이 사각형은 마우스로 드래그해서 좌우로만 움직일 수 있어야 해.
**A.**
`index.html`을 수정하여 Phaser.js 라이브러리를 추가하고 게임 화면을 구현했습니다.
- **게임 화면 구성**: `Phaser.Game` 객체를 생성해 600x600 캔버스를 만들고, 배경을 깔끔하게 디자인했습니다.
- **그리드 생성**: `this.add.graphics()`를 사용해 화면 중앙에 6x6 형태의 회색 격자(그리드)를 그렸습니다.
- **자동차(사각형) 생성**: `this.add.rectangle()`을 통해 2칸 크기(가로로 긴 형태)의 파란색 사각형(자동차)을 추가했습니다. 각 칸에 여백을 주어 보기 좋게 배치했습니다.
- **좌우 이동 로직**: 자동차에 `setInteractive`와 `setDraggable`을 적용해 마우스 인터랙션을 활성화했습니다. `drag` 이벤트에서 `Y`축은 무시하고 `X`축(좌우)만 좌표가 변하게 했으며, 그리드 밖으로 벗어나지 않도록 좌표 제한(`Phaser.Math.Clamp`)을 걸어두었습니다. 드래그가 끝나면 가까운 칸에 자석처럼 붙는(Snap) 기능도 추가했습니다.
