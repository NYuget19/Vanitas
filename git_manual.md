# GitHub 연동 기본 가이드

이 문서는 코드를 GitHub에 업로드(연동)할 때 가장 자주 사용하는 3가지 기본 명령어에 대한 설명입니다.

## 1. `git add .`
- **의미**: 현재 폴더(`.`)에 있는 변경된 모든 파일들을 다음 커밋(저장)에 포함시키기 위해 장바구니(Staging Area)에 담는 명령어입니다.
- **언제 쓰나요?**: 코드를 수정하거나 새로운 파일을 만들고 나서, 이 변경사항들을 저장할 준비가 되었을 때 사용합니다.

## 2. `git commit -m "수정 내용 메모"`
- **의미**: 장바구니에 담은 변경사항들을 하나의 '버전'으로 확정 지어 내 컴퓨터(로컬 저장소)에 저장(커밋)하는 명령어입니다. `-m` 뒤의 따옴표 안에는 어떤 내용을 수정했는지 메모를 적습니다.
- **언제 쓰나요?**: 의미 있는 작업 단위가 끝났을 때 사용합니다. 나중에 어떤 작업을 했는지 알아보기 쉽게 명확한 메모를 남기는 것이 좋습니다.

## 3. `git push origin main`
- **의미**: 내 컴퓨터에 저장된 최신 버전들(커밋들)을 인터넷에 있는 GitHub 저장소(`origin`)의 `main` 브랜치(기본 가지)로 밀어 올려서(push) 반영하는 명령어입니다.
- **언제 쓰나요?**: 로컬에서 커밋을 완료하고, 다른 사람들과 공유하거나 GitHub에 백업하고 싶을 때 마지막으로 실행합니다. (참고: 예전 프로젝트들은 `main` 대신 `master`를 사용하는 경우도 있습니다. 그럴 땐 `git push origin master`를 사용합니다.)

---

### 💡 요약: GitHub 업로드 3단계
1. 장바구니에 담기: `git add .`
2. 메모와 함께 포장하기: `git commit -m "메시지"`
3. 배송하기: `git push origin main`
